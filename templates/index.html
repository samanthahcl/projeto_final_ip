Resumo dos tópicos

a. Estruturas de seleção

Estruturas de seleção permitem que um programa tome decisões e escolha entre diferentes caminhos de execução, dependendo de condições avaliadas durante a execução. Em Python, as principais estruturas de seleção são o if, elif e else. Por exemplo, você pode verificar se um número é positivo ou negativo e executar comandos diferentes para cada caso, garantindo maior controle do fluxo do programa.

b. Estruturas de repetição

Estruturas de repetição são usadas para executar um bloco de código várias vezes, enquanto uma condição for verdadeira ou por um número determinado de iterações. As duas estruturas principais são o for e o while. O for é útil para iterar sobre sequências como listas e strings, enquanto o while executa o código enquanto uma condição lógica permanecer verdadeira. São fundamentais para automatizar tarefas repetitivas.

c. Vetores e matrizes

Vetores e matrizes são estruturas de dados que armazenam coleções de elementos. Um vetor é uma lista unidimensional, ou seja, uma sequência linear de valores. Já uma matriz é uma estrutura bidimensional (ou mais dimensões), que pode ser representada em Python como uma lista de listas, onde cada elemento é acessado por índices múltiplos (linha e coluna). São amplamente usados para manipulação de dados numéricos, imagens, tabelas e mais.

d. Funções e procedimentos

Funções e procedimentos são blocos de código que podem ser reutilizados ao longo do programa para realizar tarefas específicas. Em Python, tudo são funções, mas podemos pensar que procedimentos são funções que não retornam valor (ou retornam None). Funções recebem parâmetros, executam operações e podem retornar resultados, facilitando a organização e modularidade do código, além de evitar repetição.

e. Tratamento de exceção em Python

O tratamento de exceções permite que programas lidem com erros ou situações inesperadas durante a execução, sem parar abruptamente. Em Python, utilizamos as palavras-chave try, except, else e finally para capturar e tratar exceções. Isso garante maior robustez e controle, permitindo que o código continue funcionando ou execute ações específicas quando um erro ocorre.
